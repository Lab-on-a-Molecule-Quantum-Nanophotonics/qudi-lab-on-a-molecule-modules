Matisse laser for scanning¶

The matisse laser is intended to have its own computer with Qudi running on it. It has locally a minimal hardware running to talk to the laser, and we connect to it on the main experiment computer where the main scanner runs.

Matisse proxy¶

class MatisseCommander(*args, **kwargs)[source]¶

Bases: qudi.interface.process_control_interface.ProcessControlInterface, qudi.interface.switch_interface.SwitchInterface, qudi.interface.sampled_finite_state_interface.SampledFiniteStateInterface

A Light proxy to talk to the MatisseCommander program.

Copy and paste example configuration:

matisse:
    connect:
        wavemeter: wavemeter # required only if go to routine is used.
    config:
        address: 'localhost' # default
        port: 30000 # default
        go_to_position: # go to position parameters
            birefringent_mode: 50 # GHz (default)
            birefringent_scan_range: 1500 # steps (default)
            birefringent_scan_increment: 20 # steps (default)
            thin_etalon_scan_increment: 80 # steps (default)
            thin_etalon_signal_start: 40000 # steps (default)
            thin_etalon_signal_end: 60000 # steps (default)
            thin_etalon_set_point_scan_increment: 25 # steps (default)
            thin_etalon_set_point_scan_range: 500 # steps (default)
            thin_etalon_free_spectral_range: 260 # GHz (default)
            pzetl_scan_increment: 0.0125 # (default)
            pzetl_full_scan_range: 0.4 # (default)
            pzetl_maximal_difference: 0.3 # GHz (default)
            pzetl_lower_point: -0.85 # (default)
            pzetl_upper_point: 0.85 # (default)
            pzetl_free_spectral_range: 18 # (default)
            pzetl_relaxation: float = 1000 # ms (default)
            spzt_lower_point: 0.01 # (default)
            spzt_upper_point: 0.65 # (default)
            precision: 7 # (GHz)

prepare_bifi_scan()[source]¶

Prepare the positions scanned by the birefringent filter.

Wait for the BiFi motor to be ready, then if the current frequency is close enough to the target, skip the BiFi scan. If it is too far, reset the wavelength of the BiFi if it has not been reset for a long time, or go to the first position of the scan.

Remote matisse scanner¶

For now this is intended to use the matisse proxy on a remote computer, but could be modified to use it on the same computer if needed.

This hardware uses a state machine that is depicted below. The link are clickable and lead to the specific method documentation for the state.

Bases: qudi.interface.excitation_scanner_interface.ExcitationScannerInterface, qudi.interface.sampled_finite_state_interface.SampledFiniteStateInterface

A ExcitationScannerInterface implementation to use a Matisse laser.

Copy and paste example configuration:

matisse_scanner:
    module.Class: 'interfuse.remote_matisse_scanner.RemoteMatisseScanner'
    connect:
        input: ni_finite_sampling_input
        matisse: matisse
        matisse_sw: matisse
        wavemeter: wavemeter
    config:
        input_channels:
            - 'pfi0'
            - 'ai0'
        chunk_size: 10 # default
        max_scan_speed: 0.01 # default
